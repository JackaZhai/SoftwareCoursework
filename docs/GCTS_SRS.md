# 高校毕业生事业发展跟踪管理系统（GCTS）需求规格说明书

## 1. 引言

### 1.1 编写目的
本需求规格说明书用于明确高校毕业生事业发展跟踪管理系统（Graduate Career Tracking System, GCTS）的功能需求、性能要求、接口定义及设计约束。该文档面向系统分析人员、开发人员、测试人员、项目管理人员及最终用户，作为系统设计与开发的标准依据。

### 1.2 背景
随着教育部对高校毕业生就业质量跟踪的要求不断提高，高校需要对毕业生的职业发展进行持续性数据采集与分析。目前学校就业数据主要依赖人工表格汇总，存在数据重复、统计滞后、真实性差等问题。因此，构建一个集信息采集、问卷调查、数据分析与报告输出于一体的系统平台，是实现教育质量评估与就业指导信息化的关键。

### 1.3 系统名称和范围
- **系统名称**：高校毕业生事业发展跟踪管理系统（Graduate Career Tracking System，GCTS）。
- **主要功能范围**：
  - 毕业生个人信息与就业信息录入；
  - 辅导员审核与统计；
  - 学校就业数据分析与报告生成；
  - 问卷调查、自动通知与反馈分析；
  - 数据接口与安全保障。

## 4. 详细需求

### 4.1 功能性需求
| 编号 | 名称 | 功能描述 |
| --- | --- | --- |
| FR-01 | 用户登录与认证 | 支持毕业生、辅导员、管理员三种角色登录，接入统一身份认证系统（SSO）。 |
| FR-02 | 信息录入与更新 | 毕业生填写基本信息（姓名、学号、学院、专业、联系方式、就业单位、行业、薪资）。 |
| FR-03 | 信息审核与导出 | 辅导员查看、筛选并审核信息，支持 Excel 导出。 |
| FR-04 | 问卷管理 | 管理员创建调查问卷（毕业后半年/一年/三年），生成二维码与统计报表。 |
| FR-05 | 数据分析与统计 | 按学院、专业、年份统计就业率、薪资区间、行业比例、地域分布。 |
| FR-06 | 报表生成 | 自动生成《就业质量年度报告》，支持 PDF/Word 导出。 |
| FR-07 | 消息推送 | 向未填报或数据过期用户发送短信/邮件提醒。 |
| FR-08 | 权限与角色管理 | 按角色控制数据访问，采用 RBAC 模型。 |
| FR-09 | 日志与审计 | 系统记录操作日志，可追溯与导出。 |
| FR-10 | 外部接口 | 对接教务系统同步学籍信息，对接消息服务推送提醒。 |

### 4.2 非功能性需求
| 编号 | 类别 | 描述 |
| --- | --- | --- |
| NFR-01 | 性能 | 系统支持 500 并发用户，平均响应时间 ≤ 2 秒。 |
| NFR-02 | 可用性 | 年均可用率 ≥ 99.9%。 |
| NFR-03 | 安全性 | 使用 HTTPS 传输，AES 加密存储敏感数据。 |
| NFR-04 | 可维护性 | 模块化结构、日志化维护、支持灰度发布。 |
| NFR-05 | 可扩展性 | 支持大规模数据扩展与横向扩容。 |
| NFR-06 | 可移植性 | 可部署在容器环境，兼容主流数据库。 |
| NFR-07 | 可观测性 | 接入 Prometheus + ELK 监控告警体系。 |
